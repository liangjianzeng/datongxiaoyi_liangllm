LiangLLM APP 本地优先 多实例 实时流

Electron + FastAPI 的本地 LLM 管理控制台。一站式集成模型服务、推理调用、对话面板、实时监控与管理配置。

README.md 同步至 README.html，最后更新：2026-04-24

一、关于

LiangLLM APP 是一款面向个人/团队的本地 LLM 图形化管理工具，核心目标：

  • 一个界面 管理所有本地模型服务（llama.cpp server、后续可扩展 vLLM / Ollama / MLX 等）
  • 实时可视化 展示模型实例状态、GPU/CPU/内存占用、推理性能
  • 开箱即用对话 内置带流式响应的聊天面板，支持多实例并行
  • 零云依赖 默认完全本地运行，API 层可插拔、可离线

二、技术栈

前端

Electron 28 · Vue 3 (Vue CDN runtime) · Element Plus · CSS Modules (自定义暗色变量) · SVG 内嵌图

后端

Python 3.10+ · FastAPI 0.110+ · Pydantic v2 · 自动探测 llama-server

通信

HTTP REST + SSE (/v1/chat/completions) · IPC (Electron ↔ 主进程 ↔ backend)

服务支持

OpenAI 兼容协议，目标支持 llama.cpp server · vLLM · Ollama · MLX · DeepEP

三、页面结构

主窗口分为四大区，左栏导航，右侧顶部三 Tab（仪表盘 / 对话 / 设置/实例），下方日志区：

  • 仪表盘：实例卡片 + GPU / 系统实时指标 + 快捷操作
  • 对话：按实例管理的对话面板（带参数调节 & 流式输出）
  • 设置 / 实例：全局配置 + 实例添加编辑 + 进程管理
  • 实例卡片：每个实例显示 family / port / model / ctx / batch / 状态 / 进程

六、核心模块说明

◆ Electron 主进程

窗口创建、生命周期、IPC、环境检测（Metal 可用时启用 GPU 加速建议）。

◎ Dashboard

实例卡片（family / port / model / ctx / batch / status / process）、GPU 单卡可视化、系统指标、详情 el-dialog。

↯ Chat Panel

带模型选择、系统提示词、温度/TopP/TopK/MaxTokens 调节、流式输出 + Markdown 渲染、多实例多会话。

⚙ Settings

全局设置（日志等级、默认 prompt、API 地址）+ 实例管理（增/删/改/启/停）。

⛚ Process Manager

跨平台进程守护、任务列表、taskkill / kill 封装，支持单实例重启。

▣ GPU Monitor

nvidia-smi / Metal 设备探测，实时显存、使用率、温度、功耗。

八、功能亮点

  • 多实例管理：并行多个 llama.cpp server（或其他 OpenAI 兼容服务），不同模型、不同端口、不同进程。
  • 统一 OpenAI 兼容入口：无论底层是 llama.cpp / vLLM / Ollama / MLX，上层均暴露 /v1/chat/completions。
  • 流式对话：SSE 实时输出、Markdown 渲染、代码块高亮。
  • 参数动态调节：系统提示词、temperature、top_p、top_k、max_tokens，会话级。
  • 仪表盘 & 监控：卡片化实例 + GPU 单卡进度条 + 系统内存 / CPU。
  • 暗色主题 + 模块化组件：全部自定义 CSS 变量 + Element Plus 组件。
  • 可插拔后端：实例可配置不同 type（llama.cpp / vllm / ollama / openai 等），只需实现统一 Chat Completion 协议。

九、启动

1. 克隆

git clone <this repo>
cd LiangLLM-App

2. 后端

cd backend
python -m venv .venv
# Windows PowerShell:
.\.venv\Scripts\Activate.ps1
# macOS/Linux:
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --host 127.0.0.1 --port 8765 --reload

后端默认端口 8765，所有前端 API 请求都走它。

3. 前端（Electron）

npm install
npm start        # 启动开发模式
# 或打包：
npm run build

4. 本地模型后端（示例：llama.cpp server）

# macOS (Metal)
./llama-server -m /path/to/model.gguf --port 8080 -ngl 99

# Windows (CUDA / CPU 可选)
llama-server.exe -m D:\models\model.gguf --port 8080 -ngl 99

# vLLM（示例）
python -m vllm.entrypoints.openai.api_server --model /path/to/model --port 8081

# Ollama
ollama serve &
ollama run <model-name>

十、配置（全局 / 实例）

全局默认值保存在 Electron 用户目录 config.json，实例通过「设置」Tab 编辑。每个实例至少包含：

  • name：实例显示名
  • family：家族 / 显示组（如 Qwen2.5-7B-Instruct）
  • type：后端类型（llama.cpp / vllm / ollama / openai）
  • base_url：OpenAI 兼容地址（如 http://127.0.0.1:8080/v1）
  • model：底层模型名或 GGUF 路径
  • port：服务端口（可选）
  • ctx_size：上下文大小
  • batch_size：批大小
  • n_gpu_layers：GPU offload 层数
  • command：启动命令（可选）
  • extra_args：附加命令行参数（可选）

十一、使用说明

① 添加实例

进入「设置 / 实例管理」→ 添加实例 → 选择后端类型/端口/模型 → 保存后启动（play 按钮）。

② 查看仪表盘

首页自动拉取所有实例 + GPU + 系统指标；每张实例卡显示 family / port / model / 状态；点击详情查看进程、性能、启动参数。

③ 对话

进入「对话」Tab → 选择实例模型 → 设置系统提示词 + 温度/顶P/顶K/MaxTokens → 发送。

④ 管理进程

仪表盘卡片或实例管理中的 ▶/⏹ 按钮，底层走 process_manager（tasklist / taskkill 或 *nix kill）。

十二、接入其他模型服务

所有接入只需在后端统一暴露 OpenAI Chat Completion 兼容接口。下面为规划与实现路径：

  • llama.cpp server：http://127.0.0.1:8080/v1，已实现 + SSE 流式。
  • vLLM：OpenAI 兼容服务，支持 GPU 多卡 / Continuous Batching；在实例管理中新增 type=vllm，后端直接走同一套 Chat Completion 路由。
  • Ollama：本地管理器，ollama serve 自身提供 OpenAI 兼容 endpoint，只需在实例中配置 Ollama base_url / model 即可。
  • MLX (Apple Silicon)：mlx-lm serve 或 mlx-server，按 OpenAI 协议接入，适合 Mac mini / MacBook 本地推理。
  • DeepEP / DeepSeek 系列：推荐走 vLLM 或官方推理服务；LiangLLM APP 不绑定实现，只要暴露 OpenAI Chat Completion 即可。
  • 云厂商兼容：OpenRouter / Together AI / DeepInfra / 智谱 GLM / 百川等，配置 type=openai + api_key。
结论：任何「实现了 OpenAI Chat Completion 协议的服务」都可作为 LiangLLM APP 的后端。统一管理的关键 = 统一入口协议 + 多实例配置。
